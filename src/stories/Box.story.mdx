# Box

The `<Box />` is the building block that gives us the features of `styled-system`. It's a stripped
down version of the of the [Rebass Box](https://rebassjs.org/box), exposing only the space props,
`sx` and `variants`.

## Usage

```jsx harmony
import { Box } from '@charlietango/ui';
```

### Styling with `sx` prop

The `sx` property allows you to apply styles from a CSS object, while still being able to access the
[styled-system](https://styled-system.com/) theme values. So you can use arrays to define the
responsive values.

```jsx harmony
import { Box } from '@charlietango/ui';

const Example = () => (
  <Box
    sx={{
      p: 5,
      fontSize: 4,
      width: ['100%', null, '50%'],
      color: 'white',
      bg: 'blue',
    }}
  >
    Box
  </Box>
);
```

### Margin & padding

To make it easy to modify the `padding` and `margin` of components, the `<Box>` allows you to set
them as props. They can all be set using the shorthand naming (like `p`, `py`, `pl`, etc.) See the
API for more details.

```jsx harmony
import { Box } from '@charlietango/ui';

const Example = () => (
  <Box m={5} py={[3, 4]}>
    Space Box
  </Box>
);
```

### Making reusable components

When building components that style the `<Box />`, and expose them, you can use the `__css` prop to
set the default styling values. These will be be merged with any styles defined on `sx`, `css` or as
props, and converted it to the final class name. If you were to just use the `sx` property, then the
default styles would be overwritten the moment the consumer sets `sx`.

See other UI components for more examples, like the `<Heading>` here, that resets the `my` value:

```jsx harmony
export const Heading = props => (
  <Box as="h2" themeKey="text" variant="heading" __css={{ my: 0, ...sx }} {...props} />
);
```
