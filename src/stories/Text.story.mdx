# Text

`<Text />` should be used when creating text elements _(p, span)_. By default it will render as a
`<p>` element, but use the `as` property to change to the correct HTML tag.

## Usage

```typescript
import { Text } from '@charlietango/ui';
```

To modify the styling, you should could variants to the `text` area in the theme - preferably using
a `TextVariant` enum as the key. It might also make sense to wrap the `<Text />` inside another
component that, sets the styling. By default the `text.body` variant is used, but it will be
replaced if you set a custom value.

```typescript
export enum TextVariant {
  Paragraph = 'paragraph',
}

const theme = {
  text: {
    [TextVariant.Paragraph]: {
      fontWeight: 400,
      fontSizes: 16,
      lineHeight: 1.4,
    },
    ...
  },
};
```

You could then create a `<p>` like:

```jsx
<Text variant={TextVariant.paragraph}>
  Amor, exsul, et luna. Heu.Cum ionicis tormento peregrinationes, omnes luraes amor ferox, castus
  calcariaes.
</Text>
```

To render a `<span>`:

```jsx
<Text as="span" variant={TextVariant.inline}>
  Cur liberi ire?
</Text>
```

Text elements can be nested, just like normal HTML elements:

```jsx
<Text variant={TextVariant.paragraph}>
  Amor, exsul, et luna.
  <Text as="span" variant={TextVariant.highlight}>
    Cur liberi ire?
  </Text>
  Ionicis tormento peregrinationes, omnes luraes amor ferox, castus calcariaes.
</Text>
```
